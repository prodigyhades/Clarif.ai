ROLE-WISE EXECUTION TABLE (MEDICAL CHATBOT RAG SYSTEM)

Use Case:
User Input: "I have fever, sore throat, and fatigue"
System Goal: Retrieve medical knowledge and suggest possible conditions with disclaimers


ROLE: Data Engineer
MODULE: data_loader.py
STEPS:
1. Collect medical data (PDFs, websites, datasets)
2. Extract raw text from sources
3. Clean and normalize text
4. Split text into chunks (chunk_size ~500–1000, overlap ~100–200)

EXAMPLE:
Chunk: "Flu symptoms include fever, sore throat, fatigue, and body aches."

OUTPUT:
List of chunks:
[
  {"text": "...", "metadata": {...}}
]


ROLE: Embeddings Engineer
MODULE: embeddings.py
STEPS:
1. Take chunks from Data Engineer
2. Convert each chunk into vector embeddings using API
3. Maintain consistent format for storage

EXAMPLE:
Input: "fever, fatigue"
Output: [0.12, -0.44, 0.91, ...]

OUTPUT:
[
  {"vector": [...], "payload": {"text": "...", "metadata": {...}}}
]


ROLE: Vector DB Engineer
MODULE: vector_store.py
STEPS:
1. Store embeddings in vector database (Qdrant or similar)
2. Implement similarity search function
3. Return top-K relevant chunks based on query vector

EXAMPLE:
Query vector → returns chunks about flu

OUTPUT:
Top-K results:
[
  {"text": "...", "score": 0.92},
  {"text": "...", "score": 0.89}
]


ROLE: Retrieval Engineer
MODULE: retriever.py
STEPS:
1. Accept user query
2. Convert query → embedding
3. Perform vector search
4. (Optional) Combine with keyword search (Elasticsearch)
5. Return most relevant text chunks

EXAMPLE:
Input: "fever + throat pain"
Output: flu-related chunks

OUTPUT:
[
  "Flu symptoms include fever...",
  "Viral infections may cause sore throat..."
]


ROLE: LLM Engineer
MODULE: llm_engine.py
STEPS:
1. Design prompt template
2. Inject retrieved context
3. Add safety disclaimer
4. Call LLM API
5. Format structured response

EXAMPLE PROMPT:
"You are a medical assistant. Based on the context, suggest possible conditions. Do not provide medical advice."

OUTPUT:
"Possible conditions include flu and viral infection..."


ROLE: Frontend Engineer
MODULE: app.py (Streamlit)
STEPS:
1. Build chat interface
2. Accept user symptom input
3. Send query to backend
4. Display response + sources + disclaimer

EXAMPLE:
User enters: "fever, fatigue"
UI shows: possible conditions + explanation

OUTPUT:
Interactive UI with response


ROLE: Integration Lead
MODULE: main.py
STEPS:
1. Connect all modules together
2. Ensure correct data flow
3. Handle errors and debugging
4. Execute full pipeline

FLOW:
Input → Retrieval → LLM → Output

OUTPUT:
Fully working system


FINAL SYSTEM WORKFLOW:

User Input (Symptoms)
        ↓
Retriever (finds relevant medical context)
        ↓
LLM (generates explanation + conditions)
        ↓
Frontend (displays response)
